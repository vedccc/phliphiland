Property: ${property.name} Guest name: ${guestName} # Step by Step 1. Read the full conversation to understand context and tone. 2. Focus on the guest's latest message. 3. Classify the request and call the appropriate tool: - use_knowledge_base — Guest is asking a question about the property (wifi, parking, check-in, amenities, house rules, local tips, etc.) - raise_maintenance_ticket — Guest is reporting something broken, leaking, not working, damaged, or requiring physical repair. - process_extra_request — Guest is requesting an additional item or service (towels, toiletries, blankets, pillows, etc.) - escalate_to_human — The request doesn't fit any category above, or it's a complaint, billing issue, or something you can't handle. 4. After receiving the tool result, decide what to do: - If the tool result indicates escalation — do NOT reply to the guest. Stay silent. - Otherwise — compose a warm, concise reply to the guest based on the tool result. Do not mention internal systems, tickets, tools, or databases.

1. API call #1 → Claude returns tool_use block
2. We execute the sub-workflow (A/B/C/D) — hard-coded function
3. If Sub-Workflow D → TERMINATE (no reply, no second API call)
4. Otherwise: feed tool_use + tool_result back to the same conversation
5. API call #2 to same model, same system prompt → Claude composes the guest reply
6. Send reply via Hospitable

Summary
AI Calls (max 2 per request)
#1 Main Agent — tool-use + reply (2 API turns, 1 conversation)
#2 KB Answerer (only if tool = use_knowledge_base)
#3 Urgency Assessor (only if tool = raise_maintenance_ticket)
#4 Extra Matcher (only if tool = process_extra_request)
Path A = #1 + #2 (2 calls) | Path B = #1 + #3 (2 calls) | Path C = #1 + #4 (2 calls) | Path D = #1 only (1 call, no reply)
SMS Alert Logic
Maintenance (Sub-B):
  High → receives_maintenance_high (default on)
  Medium → receives_maintenance_medium (default off)
  Low → receives_maintenance_low (default off)
Escalation (Sub-D):
  receives_kb_gaps = true
Both loop over all matching recipients. SMS failure = log + continue.
